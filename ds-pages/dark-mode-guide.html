Como Usar em Seu Projeto
Integração rápida em Next.js 15 + React 19

1. Copiar arquivos

cp app/globals.css app/theme-provider.ts [seu-projeto]/app/

2. Envolver com ThemeProvider

// app/layout.tsx import { ThemeProvider } from './theme-provider' export default function RootLayout({ children }) { return ( <html lang="pt-BR"> <body> <ThemeProvider> {children} </ThemeProvider> </body> </html> ) }

3. Usar hook em componentes

'use client' import { useTheme } from '@/app/theme-provider' export function ThemeToggle() { const { theme, toggleTheme } = useTheme() return ( <button onClick={toggleTheme}> {theme === 'dark' ? '☀️ Light' : '🌙 Dark'} </button> ) }

4. Componentes funcionam automaticamente

// Tokens se adaptam automaticamente ao tema ativo! <button className="bg-primary-500 text-white">Click me</button>
Boas Práticas
Siga essas orientações para manter Dark Mode simples e eficiente
✅ Fazer
  • ✓ Usar CSS custom properties
  • ✓ Usar Tailwind `dark:` variant
  • ✓ Testar em navegadores reais
  • ✓ Validar contraste WCAG
  • ✓ Usar transições suaves
❌ Evitar
  • ✕ Hardcoded colors (#fff, etc)
  • ✕ Filter `invert()`
  • ✕ Media queries manuais
  • ✕ Lógica JS para mudança de cor
  • ✕ Ignorar acessibilidade
FAQ
Dúvidas frequentes sobre Dark Mode

Como forço light mode em um container?

Use a classe `light` no elemento para sobrescrever o tema escuro:

<div class="light">Este container sempre usa light mode</div>

Dark mode afeta performance?

Não. CSS custom properties são rápidas e Tailwind v4 otimiza automaticamente os seletores `:dark`.

Posso ter temas customizáveis?

Por enquanto, apenas light/dark. Temas customizáveis é future work.

Como evito FOUC (Flash of Unstyled Content)?

A função `initTheme()` é chamada no `ThemeProvider` antes de renderizar componentes. Para máxima segurança, adicione um script de inicialização inline no ``.
